Module F

Robustness & Heterogeneity

Comparing groups (e.g., Male vs. Female). We must first prove "Invariance" (they understand questions the same way) before comparing results.

1. Measurement Invariance (MICOM)

Measurement Invariance of Composites (MICOM) is required before you run Multi-Group Analysis.

Why do we need this?

Imagine you compare "Happiness" between a Child and an Adult. If the Child thinks "Happiness" means "Getting Candy" and the Adult thinks it means "Financial Security", you cannot compare their scores.

MICOM checks if the constructs have the same meaning across groups.

Group A: 5 Questions → "Trust"
= ?
Group B: 5 Questions → "Trust"
(MICOM checks if "=" holds true)

The 3-Step Procedure

SmartPLS runs these sequentially. You must pass Step 2 to continue.

Step 1: Configural Invariance

The basic model setup is the same for both groups (same number of indicators). (Always passed if data is imported correctly).

Step 2: Compositional Invariance (The Check)

Are the construct scores equal? We look at the c-value (correlation of construct scores between groups).

Step 3: Equal Means (Optional)

Do the groups have the same average scores for the construct?

Interpreting the Results

Step 2 Passed (Invariance)
The "Original Value (c)" falls within the confidence interval [5% to 95%].
5% < c < 95%
Partial Invariance
At least 80% of constructs show invariance. You can proceed with caution.
≥ 80%
Step 2 Failed
The groups interpret the constructs differently. Do NOT run MGA.
c is OUTSIDE CI

2. Multi-Group Analysis (MGA)

Once MICOM (Step 2) is passed, we can ask: "Is the path coefficient different for Group A vs. Group B?"

Comparing Path Coefficients

In the standard model, we found Quality → Satisfaction has a coefficient of 0.45.

In MGA, we split the data (e.g., by Gender):

  • Male Group: 0.40
  • Female Group: 0.60

Question: Is 0.60 significantly different from 0.40? Or is it just random chance?

Two Main Methods

1. PLS-MGA (Parametric)

Calculates the difference and tests significance using standard errors. Good for large sample sizes.

2. Permutation (Non-Parametric)

Shuffles the data randomly to create a "chance" distribution. Preferred for robustness. It works better with smaller samples.

Recommendation
Run both. If they agree, your result is rock solid.

The Decision: P-Value of Difference

Look at the "p-diff" column in the PLS-MGA results.

Significant Difference
The relationship IS different between the two groups.
p-diff < 0.05
No Difference
The groups behave similarly (e.g., Gender doesn't matter).
p-diff > 0.05
Important: Always check the Confidence Intervals. If they don't overlap (e.g., Males: 0.2-0.4, Females: 0.5-0.7), there is definitely a difference.
