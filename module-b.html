Module B

Data Preparation & Screening

SmartPLS cannot correct poor data quality. This module ensures your dataset is statistically and methodologically sound before importing into SmartPLS.

1. Initial Data Screening

Survey data typically contains missing values, careless responses, and extreme observations. These issues must be addressed at the dataset level.

Missing Values

  • Check percentage of missing values per respondent
  • Minor missing values can be imputed
  • Extensive missing values require case deletion
Acceptable Missing Data
Per respondent
< 5%

Imputation Strategy

Mean replacement within the same construct is acceptable for small amounts of missing data.

Example: Missing item replaced by the respondent’s average score on the same construct.

2. Response Quality & Careless Responding

PLS-SEM assumes respondents meaningfully engaged with survey items. Response quality must therefore be assessed explicitly.

Straight-Lining

  • Detect zero or near-zero variance across items
  • Remove respondents selecting the same scale point repeatedly

Completion Time

Extremely fast responses indicate insufficient cognitive processing.

Rule of Thumb
Remove responses completed in less than one-third of the median time.

3. Distribution, Normality & Outliers

PLS-SEM does not require multivariate normality, but extreme outliers still distort estimation and inference.

Univariate Outliers

Use standardized Z-scores to detect extreme values.

Z-Score Threshold
Extreme values
±3.29

Investigate before deleting: errors first, then genuine extremes.

4. Sample Size & Statistical Power

Sample size adequacy depends on model complexity and expected effect sizes.

10-Times Rule (Minimum)

Minimum sample size equals ten times the maximum number of structural paths pointing at any construct.

If three paths point to one construct → Minimum N = 30

G*Power Analysis (Recommended)

  • Test family: t-tests
  • Statistical test: Linear multiple regression
  • Effect size (f²): 0.15 (medium)
  • α = 0.05, Power = 0.80
Reviewer Preference
Transparent power justification

5. Common Method Bias (CMB)

When data are collected from a single survey at one time point, common method bias may inflate relationships.

Procedural Remedies

  • Anonymity assurance
  • Clear and concise item wording
  • Variation in scale anchors

Statistical Test

Full collinearity assessment using VIF values in SmartPLS.

CMB Threshold
Inner VIF values
< 3.3

6. Reverse-Coded Items

Negatively worded items must be reverse-coded to ensure consistent construct interpretation.

Reverse coding formula:
New Value = (Maximum + 1) − Original Value

Failure to reverse-code leads to misleading reliability and validity results.

7. Importing Data into SmartPLS

  1. Open SmartPLS 4
  2. Create a new project
  3. Import Excel or CSV data file
  4. Confirm rows = cases, columns = indicators
  5. Verify variable names and measurement scales
Important
Import only after completing all screening steps.
